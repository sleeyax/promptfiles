# Summary draft-note template

Fill this shape once per review and post as a non-positional draft note.

```
## Code review summary

**Files reviewed:** N — `path/a`, `path/b`, …
**Findings:** X blocker · Y concern · Z suggestion · W nit

<2–4 plain sentences in first person: what was reviewed, recommendations, effort estimate. State here if review was diff-only (no local checkout).>

> Generated by AI (`<tool>`, model: `<model>`).
```

## Rules

- **Files reviewed**: count + comma-separated list of paths in backticks. For long lists, truncate to 5 with `…and N more`.
- **Findings**: tally each severity tier; omit a tier if its count is `0` (e.g. `2 concern · 1 nit`). If zero findings overall, write `**Findings:** none`.
- **Body**: first person (`I`), no bullets, no headers, no score. Effort estimate in plain English (`a small fix`, `moderate rework`, `substantial redesign`). Mention diff-only mode here when applicable.
- **Footer**: substitute `<tool>` with the agent runtime (e.g. `Claude Code`, `Codex CLI`) and `<model>` with the model identifier in use (e.g. `claude-opus-4-7`, `gpt-5-codex`). Both stay inside the backticks. Don't leave the placeholders literal.
