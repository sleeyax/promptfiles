# Inline draft-note template

Fill this shape per inline finding. Severity is metadata only — never appears in the body.

````
**<title — ≤10 words, fact-stated, no severity word, no trailing punctuation>**

<2–4 plain sentences: (1) problem, (2) trigger conditions, (3) consequence, (4) fix. Backticks around every path, flag, identifier.>

```suggestion:-0+0
<replacement line(s), exact original indentation>
```

> [Generated by AI](https://github.com/sleeyax/promptfiles/blob/main/skills/review-mr/SKILL.md) (`<tool>`, model: `<model>`).
````

## Rules

- **Title**: ≤10 words, stated as a fact (e.g. `User input interpolated into raw SQL query`). No emoji. No trailing punctuation. No severity words.
- **Body**: 2–4 plain sentences in the order *problem → trigger → consequence → fix*. No bullets, no sub-headers, no bold inside the body. Backticks around every path, flag, function, identifier.
- **Suggestion block** (optional): drop it when the fix can't be expressed as a contiguous line replacement. Otherwise:
  - `suggestion:-N+M` replaces the anchor line plus `N` preceding and `M` following lines.
  - `-0+0` = anchor line only.
  - Preserve exact indentation (tabs vs spaces, same count) of the line(s) being replaced.
  - Replacement code only — no commentary, no `// ...` placeholders.
- **Footer**: substitute `<tool>` with the agent runtime (e.g. `Claude Code`, `Codex CLI`) and `<model>` with the model identifier in use (e.g. `claude-opus-4-7`, `gpt-5-codex`). Both stay inside the backticks. Don't leave the placeholders literal.

## `nit`-severity short form

For trivial nits, this shorter shape is allowed:

```
<single sentence, backticks around identifiers>

> [Generated by AI](https://github.com/sleeyax/promptfiles/blob/main/skills/review-mr/SKILL.md) (`<tool>`, model: `<model>`).
```

Optionally followed by a one-line `suggestion:-0+0` block.
